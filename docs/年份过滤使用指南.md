# 年份过滤使用指南

## 功能说明

年份过滤用于把数据限制在指定年份范围内，例如 `2015-2024`。

适合场景：

- 检索结果中混入 Early Access 或异常年份
- 论文方法学需要严格限定研究时间范围
- 只分析近 5 年或近 10 年趋势

## 集成到工作流中使用

```bash
python3 run_ai_workflow.py \
  --data-dir "/path/to/data" \
  --year-range 2015-2024
```

## 实际执行顺序

当设置 `--year-range` 时，工作流会先过滤原始输入，再继续后续步骤：

1. 过滤 `wos.txt`
2. 过滤 `scopus.csv`
3. 再做转换、合并、筛选、清洗和分析

这样可以尽量在源头减少异常年份带来的干扰。

## 中间输出文件

启用年份过滤后，常见中间文件包括：

- `wos_year_filtered.txt`
- `scopus_year_filtered.csv`
- `wos_year_filtered_year_filter_report.txt`

随后仍会生成常规输出，例如：

- `merged_deduplicated.txt`
- `english_only.txt`
- `Final_Version.txt`

## 使用建议

### 只分析近十年

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --year-range 2015-2024
```

### 只分析近五年

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --year-range 2020-2024
```

## 注意事项

- 参数格式必须是 `YYYY-YYYY`
- 起始年份不能大于结束年份
- 没有年份的记录默认保留
