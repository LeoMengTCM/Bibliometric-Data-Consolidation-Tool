# 机构清洗使用指南

## 作用

机构清洗用于优化 WOS 记录中的 `C3` 机构字段，减少名称变体、父子机构混杂和噪音项，方便后续在 VOSviewer、CiteSpace 等工具中做机构分析。

## 默认行为

如果你直接运行：

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data"
```

工作流默认已经启用机构清洗。

如需关闭：

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --no-cleaning
```

## 默认规则文件

当前默认规则文件为：

```text
config/institution_cleaning_rules_ultimate.json
```

你也可以手动指定：

```bash
python3 run_ai_workflow.py \
  --data-dir "/path/to/data" \
  --cleaning-config config/institution_cleaning_rules_enhanced.json
```

## 独立运行清洗器

```bash
PYTHONPATH=src python3 -m bibliometrics.standardizers.institutions \
  english_only.txt \
  english_only_cleaned.txt
```

指定自定义规则：

```bash
PYTHONPATH=src python3 -m bibliometrics.standardizers.institutions \
  english_only.txt \
  english_only_cleaned.txt \
  --config config/institution_cleaning_rules_ultimate.json
```

## 清洗后常见输出

- `Final_Version.txt`：工作流默认的清洗后文件
- `*_cleaning_report.txt`：清洗报告

## 可配置规则

规则文件主要包含：

- `noise_patterns`：噪音模式
- `standardization_rules`：名称标准化映射
- `parent_child_mapping`：父子机构映射
- `department_patterns`：部门级模式
- `merge_rules`：额外合并规则

示例：

```json
{
  "standardization_rules": {
    "chinese acad sci": "chinese academy of sciences"
  },
  "parent_child_mapping": {
    "harvard medical school": "harvard university"
  }
}
```

## 推荐工作流

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data"
```

如果你需要对已有结果再次试验不同规则：

```bash
PYTHONPATH=src python3 -m bibliometrics.standardizers.institutions \
  "/path/to/data/english_only.txt" \
  "/path/to/data/english_only_cleaned.txt" \
  --config config/institution_cleaning_rules_ultimate.json
```

## 经验建议

- 优先使用默认 `ultimate` 规则
- 只有在你明确知道某些机构名称仍需手工归并时，再新增映射
- 修改规则后，最好重新跑一次清洗并查看 `*_cleaning_report.txt`
