# 数据准备说明

## 推荐的数据准备原则

为了把这个项目真正用成**以 WOS 为标准的 Scopus→WOS 转换与整合系统**，最推荐的输入方式是：

- `wos.txt` 与 `scopus.csv` 来自**同一检索主题或同一课题集合**
- 两边尽量使用接近的检索式和时间范围
- 尽量保留足够多的 WOS / Scopus 重复文献

这样做的价值不只是为了合并，更是为了让系统能通过重复文献去校准转换规则，尤其是 `C3` 的选择与补充逻辑。

## 必需的输入文件

运行主工作流前，请把以下两个文件放到同一个数据目录中：

```text
/你的数据目录/
├── wos.txt
└── scopus.csv
```

仓库自带的 `Example/` 就是一个可复现的标准示例：

```text
Example/
├── wos.txt
└── scopus.csv
```

## 1. `wos.txt`

**来源**：Web of Science

**导出建议**：

1. 完成检索后选择文献
2. 点击“导出” → “其他文件格式”
3. 文件格式选择“纯文本”
4. 记录内容选择“全记录与引用的参考文献”
5. 保存后命名为 `wos.txt`

**文件头示例**：

```text
FN Clarivate Analytics Web of Science
VR 1.0
PT J
...
```

## 2. `scopus.csv`

**来源**：Scopus

**导出建议**：

1. 完成检索后选择文献
2. 点击“Export” → “CSV”
3. 导出范围尽量选择 `All available information`
4. 保存后命名为 `scopus.csv`

**强烈建议包含的字段**：

- Authors
- Title
- Year
- Source title
- Affiliations
- DOI
- References
- Author full names
- Correspondence address（如果可选）

Scopus 原始 affiliation 越完整，当前 `C3` 的保守恢复逻辑就越有依据。

## 文件放置原则

推荐为每个课题单独建一个数据目录，例如：

```text
/Users/yourname/Bibliometrics/Project_A/
├── wos.txt
└── scopus.csv
```

然后从项目根目录运行：

```bash
python3 run_ai_workflow.py --data-dir "/Users/yourname/Bibliometrics/Project_A"
```

## 基础检查

### 检查 `wos.txt`

```bash
head -10 wos.txt
```

通常应能看到：

- 第一行是 `FN ...`
- 第二行是 `VR ...`
- 后续包含 `PT`、`AU`、`TI`、`SO` 等字段

### 检查 `scopus.csv`

```bash
head -5 scopus.csv
```

通常应能看到：

- 第一行是 CSV 表头
- 后续行为实际记录
- 表头中最好能看到 affiliation、DOI、references 等关键列

## 推荐的首次运行

建议先跑仓库自带示例，确认流程和输出：

```bash
python3 run_ai_workflow.py --data-dir Example --no-ai
```

确认没问题后，再跑你自己的数据目录。

## 常见问题

### 为什么主工作流要求同时提供 WOS 和 Scopus？

因为当前系统不只是“把 Scopus 转出来”，还要做：

- WOS 主标准对齐
- 重复对校准
- WOS-first 合并
- 重复文献验证

如果没有 WOS，系统仍可做基础转换，但会失去当前这套校准与验证优势。

### 文件名不是 `wos.txt` / `scopus.csv` 怎么办？

最简单稳妥的方式是重命名为这两个标准文件名。当前主工作流默认按这两个文件名识别输入。
