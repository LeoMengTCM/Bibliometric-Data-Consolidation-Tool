# 使用指南

## 项目定位

当前工作流应被理解为一个**以 WOS 为标准的 Scopus→WOS 转换与整合系统**。它的重点不是泛化清洗，而是：

- 把 Scopus 尽量对齐到 WOS 风格
- 用当前 WOS 输入做校准
- 做好 WOS / Scopus 去重与合并
- 让 Scopus 独有记录尽量自然地融入 WOS 语料

## 入口说明

项目支持两个等价入口：

- `python3 run_ai_workflow.py ...`
- `python3 scripts/run_workflow.py ...`

推荐优先使用 `run_ai_workflow.py`，命令更短，也更适合作为文档示例。

## 基本用法

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data"
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--data-dir` | 数据目录，必须包含 `wos.txt` 和 `scopus.csv` | 无 |
| `--language` | 语言筛选目标 | `English` |
| `--no-ai` | 禁用 AI 分支；本地 WOS 对齐逻辑仍保留 | 关闭 |
| `--no-cleaning` | 禁用机构清洗 | 关闭 |
| `--cleaning-config` | 指定机构清洗规则文件 | `config/institution_cleaning_rules_ultimate.json` |
| `--year-range` | 年份范围，如 `2015-2024` | 不限制 |
| `--log-level` | 日志级别，可选 `DEBUG` / `INFO` / `WARNING` / `ERROR` | `INFO` |

## 推荐运行方式

### 1. 先跑仓库自带示例

```bash
python3 run_ai_workflow.py --data-dir Example --no-ai
```

这一步适合先确认：

- CLI 是否正常
- 输入布局是否正确
- 本地 WOS 对齐与 `C3` 逻辑是否按预期工作

### 2. 再跑你自己的数据目录

```bash
python3 run_ai_workflow.py --data-dir "/path/to/your-data"
```

## 常用示例

筛选中文文献：

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --language Chinese
```

关闭 AI 分支：

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --no-ai
```

关闭机构清洗：

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --no-cleaning
```

限制年份范围：

```bash
python3 run_ai_workflow.py --data-dir "/path/to/data" --year-range 2015-2024
```

指定清洗规则：

```bash
python3 run_ai_workflow.py \
  --data-dir "/path/to/data" \
  --cleaning-config config/institution_cleaning_rules_ultimate.json
```

查看帮助：

```bash
python3 run_ai_workflow.py --help
```

## 实际工作流顺序

默认情况下，流程大致为：

1. 检查输入文件
2. 如有需要，执行年份过滤
3. 将 Scopus 转为 WOS 风格文本
4. 用当前 WOS 语料校准转换结果，尤其是组织字段与 `C3`
5. 若启用 AI，执行 AI 标准化与机构补全
6. 合并 WOS 与 Scopus 并去重
7. 按语言筛选
8. 按规则清洗机构名称
9. 统计分析
10. 生成图表与报告

## 输出文件说明

常见输出包括：

- `scopus_converted_to_wos.txt`
- `scopus_enriched.txt`
- `merged_deduplicated.txt`
- `english_only.txt` 或其他语言筛选结果
- `Final_Version.txt`
- `*_analysis_report.txt`
- `ai_workflow_report.txt`
- `Figures and Tables/`

如果使用 `--no-cleaning`，最终分析可能直接基于语言筛选后的文件，而不一定生成 `Final_Version.txt`。

## 关于 `--no-ai`

当前 `--no-ai` 的真实行为是：

- 跳过 AI WOS 风格标准化
- 跳过 AI 机构补全
- 保留本地转换、WOS 语料校准、`C3` 恢复、合并、筛选、清洗和分析流程

也就是说，`--no-ai` 并不意味着“只剩最原始转换”；当前最关键的 WOS 对齐逻辑仍然存在。

## 常见问题

### 这是纯靠算法，不靠数据库吗？

不是“纯泛化算法”，也不是“主要靠外部数据库查表”。

更准确地说，当前系统是：

- 本地规则驱动
- 以当前输入的 WOS 语料做校准
- 结合原始 Scopus affiliation 证据做保守补充

### 没配置 API 可以运行吗？

可以。请使用 `--no-ai`，这样仍能保留当前最关键的本地 WOS 对齐逻辑。

### 只有 Scopus，没有 WOS，可以跑吗？

主工作流的推荐用法是同时提供 `wos.txt` 和 `scopus.csv`。如果缺少 WOS，你仍然可以做 Scopus 转换，但会失去当前这套 WOS 校准、重复对验证和 WOS-first 合并的核心优势。
